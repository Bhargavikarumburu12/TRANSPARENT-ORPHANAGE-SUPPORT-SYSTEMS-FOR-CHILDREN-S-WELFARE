Methodology
• The project follows a structured development methodology that integrates system design, government data integration, web-based interface development, and secure fund tracking mechanisms. The overall approach ensures transparency, accountability, and real-time monitoring of donations made to orphanages.
1.Requirement Analysis Identified core user roles: Donor, Orphanage Administrator, and Government Medical Authority. Gathered system requirements focusing on transparency, fund traceability, and real-time reporting.
2. System Design Created system architecture diagrams including user flow, data flow, and role-based access control. Designed block diagrams outlining interactions between components like donor portal, orphanage request panel, and
government validation
3. Technology Stack
Chose web technologies for frontend (HTML, CSS, JS) and backend (Node.js/PHP/Python).
a. Implemented AES encryption for secure transactions and data privacy.
b. Database setup with MySQL/MongoDB for storing user data, donation logs, and request records.
4.Integration with Government Healthcare APIs
Simulated integration with government health schemes to verify requests and authorize services/products for orphanage children with medical needs.
5.Secure Transaction and Monitoring System
Introduced encrypted tracking from donation to delivery using unique IDs. Alerts and notifications sent to donors at each step (approval, dispatch, delivery).
6.Data Logging and Location Mapping
Captured geo-coordinates of each request/delivery using Google Maps or GPS APIs. Stored logs for transparency audits and report generation.
7.Testing and Validation
Conducted unit testing of modules and full system integration tests.
Simulated data to ensure accurate functioning of verification and traceability features. 
8.Deployment and Demonstration
Deployed the system prototype on a local server or hosting platform. Demonstrated end-to-end flow from donation to verified delivery.
